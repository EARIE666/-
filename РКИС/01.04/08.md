## Задание 8. Анализ существующих нотаций для моделирования баз данных

| Нотация | Тип | Ключевые элементы | Преимущества | Область применения |
| :--- | :--- | :--- | :--- | :--- |
| Нотация Чена (Peter Chen) | Концептуальная (ER) | Прямоугольник — сущность<br>Ромб — связь<br>Овал — атрибут | Наглядно разделяет сущности и отношения<br>Понятна нетехническим специалистам<br>Подходит для согласования с заказчиком | Концептуальное моделирование<br>Обсуждение структуры с бизнес-аналитиками<br>Обучение |
| Воронья лапка (Crow's Foot) | Логическая / Физическая | Линии с вилочками на концах показывают мощность связи (1:1, 1:M, M:N) | Самая популярная нотация в инструментах<br>Интуитивно понятное отображение связей<br>Легко транслируется в SQL-код | Реляционное моделирование<br>Создание физических схем<br>MySQL Workbench, DBeaver |
| IDEF1X | Физическая (реляционная) | Строгие правила для первичных ключей, доменов, классификаций | Стандарт США для государственных проектов<br>Максимальная формализация и строгость<br>Подходит для крупных корпоративных систем | Государственные информационные системы<br>Крупные корпоративные проекты<br>Erwin Data Modeler |
| UML (Class Diagram) | Объектно-ориентированная | Классы, атрибуты, методы, ассоциации, наследование | Единый язык для ООП и БД<br>Поддержка наследования<br>Широкая поддержка в IDE | Проектирование сложных систем (Java, C#, Python)<br>Интеграция БД с объектной моделью приложения |
